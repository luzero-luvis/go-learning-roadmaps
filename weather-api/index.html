Key Design Patterns Used

Dependency Injection

  • Config passed to components
  • Cache passed to services
  • Services passed to handlers
  • Easy testing via mocking

Interface Segregation

  • Cache interface abstracts Redis
  • WeatherService interface defined
  • Loose coupling between layers
  • Easy to swap implementations

Service Layer Pattern

  • Business logic in services/
  • HTTP handling in handlers/
  • Clean separation of concerns
  • Reusable service logic

Cache-Aside Pattern

  • Check cache first
  • Fetch from API on miss
  • Store result in cache
  • Set TTL (12 hours)

üöÄ Build from Scratch - Step by Step

Command-by-Command Guide:

  1. Initialize Go module:
    go mod init github.com/yourusername/weather-api
  2. Install dependencies:
    go get github.com/gin-gonic/gin
    go get github.com/go-redis/redis/v8
    go get github.com/spf13/viper
    go get github.com/didip/tollbooth
    go get github.com/swaggo/gin-swagger
    go get github.com/swaggo/swag
  3. Create project structure:
    mkdir -p cmd internal/{api,cache,config,dto,handlers,middleware,services} docs
  4. Implement files in this order:
    1. config/config.go - Load config
    2. dto/data_transfer_objects.go - Define structs
    3. cache/cache.go - Redis interface + implementation
    4. services/weather_service.go - Business logic
    5. handlers/weather_handler.go - HTTP handlers
    6. middleware/limiter.go - Rate limiting
    7. api/api.go - Router setup
    8. cmd/main.go - Entry point
  5. Generate Swagger docs:
    swag init
  6. Create config.json and docker files
  7. Test locally:
    go run cmd/main.go
  8. Build Docker image:
    docker-compose build
